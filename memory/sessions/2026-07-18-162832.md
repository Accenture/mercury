- **Root cause — AI-grammar gap.** The AI-facing docs (`command-reference.md`, `minigraph-commands.json`)
  *listed* `graph.math`'s statement keywords (`COMPUTE`/`IF`/`MAPPING`/`EXECUTE`/`RESET`, `NEXT:`/`BEGIN`/
  `END`) but gave **no statement syntax**. The real decision grammar is the triad `IF: <bool>` /
  `THEN: <node|next>` / `ELSE: <node|next>`; `NEXT:` takes a node name (not a relation), `BEGIN`/`END`
  are `for_each` delimiters. The full grammar lived only in the interactive `help graph-math.md`, which
  a canonical-docs-only agent won't read — so the keyword inventory *without* syntax actively invited a
  wrong-but-plausible guess that fails silently (POST returns 200; the parser error only reaches the WS
  console).

- **Doc fix (Java canonical).** Documented the `graph.math`/`graph.js` **statement grammar** — a new
  section in `command-reference.md` (five statement types + the IF/THEN/ELSE construct with a worked
  example + the NEXT:/BEGIN/END/COMPUTE-result clarifications) and a `statement_syntax`+`notes` block in
  `minigraph-commands.json`. **Landed via PR (public upstream, not direct-to-main):**
  [Accenture/mercury-composable#187](https://github.com/Accenture/mercury-composable/pull/187).
  (Rust repo carries no such guides — its `help graph-math.md` already had the full grammar.)
