## Decisions

- **Join-barrier completion semantics: "success-only and current"** (maintainer + companion
  tightening, 2026-07-19) ‚Äî a failed exception-routed branch never satisfies the barrier; a
  reset branch stops satisfying it until it re-executes successfully. Chosen over the old
  "ran once" semantics whose failure mode was a silently incomplete output; the new failure
  mode for a permanently failing branch is a visible timeout.
