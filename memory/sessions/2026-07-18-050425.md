## What happened

**Increment 31 — full declarative dev-gating for the Playground (Java parity).** Resolution of
loose-end #1 from the tutorial-3 investigation: in Java, `@OptionalService` applies to `@PreLoad`,
`@WebSocketService`, `@BeforeApplication`, and `@MainApplication`, and the minigraph Playground is a
conditional (`app.env=dev`) load. Increment 30 only gated `#[preload]`; this increment extends the
reach and converts the Playground from programmatic registration to declarative annotations.
Maintainer chose **"Full declarative refactor"** with **"Per-app config sets app.env=dev"** for the
default.

**Phase 1 — macro extension** (platform-core, commit `d582123`):
- `#[optional_service]` (marker + `optional_service = "…"` param) is now consumed by
  `#[websocket_service]`, `#[before_application]`, and `#[main_application]` as well as `#[preload]`.
  Added `strip_optional_service`/`optional_service_expr` helpers in `platform-macros`.
- `WsServiceEntry` / `BeforeAppEntry` / `MainAppEntry` each gained `optional_service`.
- `AppStarter::run` gates the ws-service loop, before-app loop, preload loop, and main-app loop on
  `feature::is_required`, logging `Skip optional …` and skipping the registration when the condition
  fails. Moved `AppConfigReader::get_instance()` up so it's available before the before-app loop.

**Phase 2 — declarative Playground** (knowledge-graph, commit `448f125`):
- Retired the programmatic `PlaygroundLoader`. Every Playground service is now a declarative struct:
  `GetIndexHtml` (`get.index.html`, **always-on** — Java's non-optional `GetIndexHtml`),
  `GraphCommandServiceFn`/`GraphCommandSingleton`, `GraphTravelerService` (+ `#[zero_tracing]`
  `#[event_interceptor]`), `GetWsHtml`, `PostCompanionCommand`, `UploadJsonContent`,
  `UploadMockContent`, `ShowGraphModel`, `GetLiveGraph`, `InspectStateMachine` — all
  `#[preload]` + `#[optional_service("app.env=dev")]` except the always-on home page; plus a
  `PlaygroundHousekeeping` `#[before_application(sequence = 8)]` `#[optional_service("app.env=dev")]`.
- `ws_ui.rs`: `GraphUserInterface` / `JsonPathHandler` now carry
  `#[websocket_service(…)]` + `#[optional_service("app.env=dev")]` (was programmatic
  `register_ws_service`). Stale module/rest/main doc comments referencing the retired loader updated.
- **Parity note:** this makes Rust *more* faithful, not less — Java registers these via annotations;
  its `PlaygroundLoader` (`@MainApplication(sequence=8)`) is only the `@FetchFeature` loader, never
  the service registrar. Our former programmatic loader was the divergence.
- `graph_runtime` now runs `app.env: dev` and inherits the engine's dev-gated mocks; removed its
  local `v1.hello.task`/`mock.mdm.profile`/`mock.account.details` copies + `serve_mock_json` (they
  would collide with the now-registering engine mocks). **Supersedes increment 30's "graph_runtime
  runs with app.env unset"** arrangement.
- **`app.env` is env-overridable**: the example app's `application.yml` now uses `${APP_ENV:dev}`
  (Java parity; config reader supports `${VAR:default}`). Default dev; `APP_ENV=prod` disables the
  whole Playground.
