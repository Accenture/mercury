- `platform.rs` reworked — **manager-worker dispatch** (ServiceQueue/WorkerHandler port):
  per route one manager task (owns the elastic queue + ready FIFO + uniqueness set +
  `buffering` starting true) and N pull-based workers (announce Ready → one event → process
  → repeat; capacity-1 event channels). Manager mailbox **bounded**
  (`elastic.queue.dispatch.mailbox.size`, default 1024, min 20) — full mailbox suspends
  senders (back-pressure, not drops). Boxed mailbox Event variant (clippy). Release via
  Notify → manager destroys the queue, workers exit on channel close.
- `post_office.rs` — `send` now routes through `Platform::deliver` (manager mailbox).
